# Processed Data

This folder contains cleaned and preprocessed datasets derived from the original raw data.

## Purpose

The processed data is designed to be **analysis-ready**, enabling statistical modelling and exploratory analysis without the need to repeat initial data cleaning steps.

## Processing Overview

The raw data undergoes a structured pipeline implemented in the `scripts/` folder, including:

* Filtering for relevant sampling methods (e.g. ANAA, ANLA, ANLE)
* Removal of duplicate observations (site × date level)
* Standardisation of data types across multiple datasets
* Temporal filtering to ensure consistency in data collection practices (post-1990)
* Integration of multiple data sources (site, metrics, taxa, WHPT)
* Classification of observations into:

  * **Categorical (bin-based) data**
  * **Numerical (count-based) data**
* Feature engineering (e.g. seasonal indicators)
* Construction of modelling-ready datasets

## Data Characteristics

The processed datasets address several challenges present in the raw data:

* Mixed measurement formats (categorical abundance bands and numeric counts)
* Missing and inconsistent values
* Changes in sampling methodology over time
* Implicit absence (non-recorded zeros)

## Contents

This folder may include:

* Cleaned sample datasets (for demonstration purposes)
* Intermediate datasets used for modelling
* Final structured tables for statistical analysis

> Note: Full processed datasets are not included due to size constraints.

## Reproducibility

All processed data can be regenerated by running the scripts in the `scripts/` folder in sequence. This ensures transparency and reproducibility of the analysis.

## Notes

* The preprocessing pipeline is designed to preserve the integrity of the original data while making it suitable for statistical modelling.
* Special care has been taken to correctly handle mixed data types and represent uncertainty in the data appropriately.
